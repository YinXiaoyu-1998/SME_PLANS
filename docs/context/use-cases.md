# 企业资料中枢 Use Cases

本文档描述员工使用 Codex、小龙虾或其他通用 AI agent 时，和“企业资料中枢”互动的常见场景。这里不讨论底层技术，只从员工日常工作角度说明：它能帮大家少找文件、少重复整理、少丢信息，并让不同员工的 AI 工具围绕同一套资料工作。

## 一句话理解

企业资料中枢不是一个替大家分析经营的“大系统”，而是企业内部的“统一资料柜”。员工手里的 AI agent 可以从这里找资料、交回报告、查询已有 skill、沉淀会议和经营信息。

员工仍然使用自己熟悉的 AI 工具完成分析和写作；企业资料中枢负责让资料不散、来源可查、权限可控、结果可复用。

## 常见使用方式

| 场景 | 员工会怎么说 | AI agent 会做什么 | 资料中枢的价值 |
|---|---|---|---|
| 找经营数据 | “帮我找保利店上周的美团数据” | 在员工有权限的资料里查找相关文件或数据 | 不用到处问人、翻群、翻电脑 |
| 做门店周报 | “用上周数据生成保利店周报” | 找数据、调用周报 skill、生成报告 | 报告口径和资料来源更统一 |
| 做菜单分析 | “帮我跑一下 5A 菜单分析” | 查询可用 skill，找到所需输入资料 | 员工不用记 skill 在哪里 |
| 上传新资料 | “把这份美团导出表放到资料中枢” | 推荐标签，等待员工确认后上传 | 新资料以后能被自己和有权限同事找到 |
| 分享给同事 | “把这份报告也给李杰看” | 给资料增加李杰的个人标签 | 不用反复发微信、邮件、网盘链接 |
| 查历史报告 | “找一下上个月做过的菜单毛利报告” | 搜索员工有权限的历史报告 | 避免重复做同一份分析 |
| 沉淀会议纪要 | “把今天会议纪要存进去，给管理层可见” | 上传纪要并打上相关标签 | 会议结论以后可搜索、可引用 |
| 找操作手册 | “有没有美团数据下载 SOP？” | 查询公开或有权限的管理知识 | 新员工学习成本降低 |
| 查看可用 skill | “现在麦家有哪些可用 skill？” | 查询 Skill 目录并展示用法示例 | 员工知道 AI 能做哪些固定工作 |
| 交接资料 | “我离职/调岗前，把我负责的资料整理一下” | 列出个人标签下的资料，辅助转交 | 资料不跟着个人电脑消失 |

## Use Case 1：让 AI agent 帮忙找资料

**用户**

> 帮我找一下保利店上周的经营数据，最好是美团导出的那份。

**AI agent 的行为**

1. 去企业资料中枢里找“保利店”“上周”“美团经营数据”相关资料。
2. 只返回当前员工有权限看的资料。
3. 如果找到多份，按时间、来源、文件名帮员工列出来。
4. 员工选择后，AI agent 再读取文件并继续分析。

**对员工的价值**

员工不用知道文件存在谁电脑里、群里发过几次、文件名叫什么。只要说清楚门店和时间，AI agent 就能从统一位置找。

## Use Case 2：上传资料并自动沉淀

**用户**

> 这是刚下载的保利店 6 月美团数据，帮我放到资料中枢里。

**AI agent 的行为**

1. 识别文件大概属于“保利店”“美团数据”“6 月”。
2. 给出建议标签，例如“保利店”“张纵”。
3. 让员工确认标签。
4. 上传资料。
5. 等资料处理完成后，之后就可以被有权限的人搜索和使用。

**对员工的价值**

员工不用手动维护复杂目录。只要确认资料属于谁，后面就能被对应员工和 AI agent 找到。

## Use Case 3：上传后先不可见，处理好再使用

**用户**

> 我刚上传的表为什么还搜不到？

**AI agent 的回答**

> 这份资料还在处理中。处理完成后才会进入可查询状态。现在不会被用于分析，避免拿到半成品数据。

**对员工的价值**

系统不会把刚上传但还没整理好的资料混进分析里。员工看到的资料默认是已经准备好的，减少误用。

## Use Case 4：生成门店周报

**用户**

> 帮我生成保利店上周周报，重点看销售趋势、异常、差评和下周动作。

**AI agent 的行为**

1. 从企业资料中枢找保利店上周相关经营数据、评价数据、历史周报。
2. 查询是否有“周报”相关 skill。
3. 按固定口径生成周报。
4. 生成后询问是否要把报告放回资料中枢。
5. 员工确认后，报告被打上“保利店”和上传者个人标签。

**对员工的价值**

周报不再每周从零开始。数据、历史报告和常用口径都能复用，员工只需要检查结论和补充业务判断。

## Use Case 5：运行菜单分析 skill

**用户**

> 现在有什么菜单分析工具可以用？我想看最近三个月菜品毛利。

**AI agent 的行为**

1. 查询企业资料中枢里的 Skill 目录。
2. 找到 5A 菜单分析或菜单毛利分析相关 skill。
3. 展示这个 skill 需要什么输入资料、会产出什么报告、怎么使用。
4. 帮员工找最近三个月的菜单和销售相关数据。
5. 调用员工本地或当前 AI 工具可用的 skill 来生成报告。

**对员工的价值**

员工不用记住 skill 名字、安装位置、使用参数。只需要说“我想分析菜单”，AI agent 就能找到合适能力。

## Use Case 6：把报告分享给指定同事

**用户**

> 这份保利店菜单分析报告也给李杰看一下。

**AI agent 的行为**

1. 找到这份报告。
2. 确认当前员工有权限修改分享范围。
3. 给资料增加“李杰”的个人标签。
4. 告诉员工：李杰之后可以通过自己的 AI agent 搜到这份报告。

**对员工的价值**

不用把文件复制一份再发来发去。资料仍然在统一位置，谁能看由标签控制。

## Use Case 7：保护无权限资料

**用户**

> 帮我找苏州街店上个月的绩效资料。

**AI agent 的行为**

如果当前员工没有相关标签，AI agent 不会显示“有但你无权查看”，也不会暴露文件名或数量，只会说没有找到可访问资料。

**对员工的价值**

员工只会看到自己该看的资料。系统不会因为搜索而泄露其他门店或其他人的敏感信息。

## Use Case 8：沉淀会议纪要和管理经验

**用户**

> 把今天运营会纪要存一下，里面关于差评处理的话术以后还会用。

**AI agent 的行为**

1. 整理会议纪要。
2. 建议把它归为“管理知识”。
3. 推荐相关标签，例如“全员可见”或指定管理层个人标签。
4. 员工确认后上传。
5. 以后有人问“差评处理话术”，AI agent 可以找到这份纪要。

**对员工的价值**

老板和管理层反复讲的经验不会只停留在会议里。新员工、门店经理、运营同事都能在需要时重新找到。

## Use Case 9：记录经营事件，帮助以后解释数据

**用户**

> 记一下，保利店 6 月 12 日到 6 月 16 日装修，营业额下降不要直接算经营问题。

**AI agent 的行为**

1. 把这条信息作为经营事件记录。
2. 打上“保利店”标签。
3. 之后做周报、月报或异常分析时，AI agent 可以把这个事件作为解释线索。

**对员工的价值**

很多经营数据异常不是单靠数字能解释的。把装修、闭店、活动、天气、人员变动这些事实沉淀下来，后续分析会更像“懂业务的人”。

## Use Case 10：归档旧资料，而不是彻底删除

**用户**

> 这份表下载错了，不要再让大家用了。

**AI agent 的行为**

1. 找到这份资料。
2. 帮员工或管理员把它归档。
3. 归档后，普通搜索和分析不会再使用它。
4. 原始文件仍然保留，管理员以后可以追溯。

**对员工的价值**

错误资料不会继续影响分析，但也不会因为误删而完全找不回。

## Use Case 11：员工交接和资料转移

**用户**

> 帮我列一下我名下最近三个月上传过的报告，哪些需要交接给李杰。

**AI agent 的行为**

1. 查询当前员工个人标签下的资料。
2. 按门店、时间、类型列出。
3. 员工选择需要交接的资料。
4. 给这些资料增加李杰个人标签，或增加相关门店标签。

**对员工的价值**

资料不会只存在个人电脑、个人聊天记录或个人网盘里。调岗、离职、换负责人时，交接更清楚。

## Use Case 12：管理员处理失败资料

**管理员**

> 看一下最近有没有上传失败或处理失败的资料。

**AI agent 或后台的行为**

1. 列出处理失败的资料。
2. 显示失败原因摘要，例如文件格式不支持、文件损坏、标签未确认。
3. 管理员决定重新处理、补标签、归档或联系上传者。

**对企业的价值**

资料处理问题不会悄悄堆积。管理员可以及时发现和修复，让资料中枢保持可靠。

## Use Case 13：低技术员工学习可用能力

**用户**

> 我不会写复杂提示词，你告诉我现在能做哪些事。

**AI agent 的行为**

1. 查询 Skill 目录和常见使用说明。
2. 用自然语言列出当前可用能力，例如：
   - 生成门店周报
   - 分析菜单毛利
   - 汇总群消息
   - 整理会议纪要
   - 查询历史报告
3. 给出可直接复制的提问示例。

**对员工的价值**

员工不用懂“工具配置”“参数”“数据入口”。他只需要看到自己可以怎么问、能得到什么结果。

## Use Case 14：跨 AI 工具继续工作

**用户**

> 我昨天用小龙虾生成了一份保利店周报，今天想用 Codex 继续改成 PPT。

**AI agent 的行为**

1. Codex 从企业资料中枢找到昨天那份周报。
2. 读取报告内容。
3. 帮用户改写成 PPT 提纲或演示材料。
4. 用户确认后，再把新的 PPT 放回资料中枢。

**对员工的价值**

不同 AI 工具不再各做各的。只要资料和产物都回到企业资料中枢，员工可以在不同工具之间接着干活。

## Use Case 15：找不到资料时，指导下一步

**用户**

> 帮我找保利店 6 月会员消费数据。

**AI agent 的行为**

如果没有找到可访问资料，AI agent 可以提示：

> 当前资料中枢里没有找到你可访问的保利店 6 月会员消费数据。你可以先从对应后台下载，上传时选择“保利店”标签。上传处理完成后，我就能继续帮你分析。

**对员工的价值**

“找不到”不是死胡同。AI agent 会告诉员工下一步该补什么资料。

## 哪些事情不由企业资料中枢直接做

为了避免误解，企业资料中枢第一阶段不直接做这些事：

- 不替员工自动做复杂经营分析。
- 不自己生成周报、PPT 或 dashboard。
- 不运行菜单分析、经营诊断等 skill。
- 不替员工决定资料该给谁看。
- 不绕过第三方平台直接拿数据库。

这些事情仍然由员工手里的 AI agent、业务 skill 或未来独立服务完成。企业资料中枢的任务是把资料、结果、标签和可用能力统一管理好。

## 最有价值的日常收益

1. 员工不用到处找文件。
2. AI agent 不用每次从零开始。
3. 报告和分析结果能沉淀复用。
4. 不同 AI 工具可以围绕同一套资料协作。
5. 资料分享更清楚，不靠反复转发文件。
6. 离职、调岗、交接时资料不丢。
7. 管理层经验、会议结论、操作 SOP 能持续积累。
8. 员工只看到自己该看的资料，减少误看和误传。
